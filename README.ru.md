# Hydra

[English](README.md) · **Русский** · [中文](README.zh.md)

**Многопоточный менеджер загрузок для macOS с пробросом браузерной сессии.**

Качает файлы в несколько потоков (HTTP Range) и пробрасывает вашу авторизованную
сессию из браузера во все потоки — поэтому скачивает быстрее и то, что доступно
только вам. Живёт в строке меню, дружится с браузером сам, без копирования id и
правки конфигов.

macOS 13+ · Swift · нативный SwiftUI/AppKit

## Возможности

- **Многопоточная загрузка** — файл бьётся на блоки, пул соединений тянет их
  параллельно (HTTP Range). Скорость, ETA и число потоков — в реальном времени.
- **Проброс сессии** — расширение снимает `Cookie`, `User-Agent`, `Referer` для
  ссылки и повторяет их **в каждом** потоке. Скачивается то, что за авторизацией.
- **Пауза / резюм, переживающий перезапуск** — карта готовых блоков пишется на
  диск; закрыл приложение с недокачанным → открыл → продолжает с того же места.
- **Дефраг-сетка блоков** — видно весь файл: что готово, что качается сейчас,
  что впереди, и «X / Y блоков».
- **Очередь** — лимит одновременных загрузок, остальные ждут; приоритет, пауза всех.
- **История** — завершённые сохраняются между запусками, поиск и «Показать в Finder».
- **Перехват из браузера** — авто-перехват по правилам (размер, типы файлов) или
  «Скачать через Hydra» в контекстном меню.
- **Плавающее окно-приёмник** — поверх всех окон; перетащи ссылку откуда угодно.
- **Нативные уведомления** — завершение, ошибка (с «Повторить»), требуется вход.
- **Нативный язык macOS** — системный акцент, материалы/вибрантность, SF, тёмная/светлая тема.
- **Локализация** — английский, русский, китайский; переключатель в Настройки → Система → Язык.

## Как это работает

```
Браузер ──(перехват ссылки + сессия)──▶ Расширение
                                            │ native messaging
                                            ▼
                                      hydra-host  ──(Unix socket /tmp/hydra.sock)──▶ Hydra.app
                                       (фолбэк: качает сам, если app не запущена)        │
                                                                                          ▼
                                                                                   DownloadCore
                                                                          probe → блоки → N потоков → сборка
```

- **Авто-связка без участия юзера.** Chrome-id зафиксирован ключом в манифесте
  (детерминированный `hfdmeoleepighofjiookfjcjekoopaim`), а Hydra.app при запуске
  сама прописывает native-messaging-хост во все браузеры. Копировать id и звать
  `install.sh` не нужно.
- **Два движка ядра:** `ResumableDownload` (блочный, пауза/резюм — для приложения)
  и `Downloader` (непрерывные куски, fire-and-forget — для CLI и фолбэка хоста).
  Оба покрыты байт-в-байт тестами.

## Установка

1. Скачайте `Hydra.app` из [Releases](../../releases), перетащите в `/Applications`,
   запустите один раз — она сама пропишет хост во все установленные браузеры.
2. Поставьте расширение своего браузера:
   - **Chrome / Brave / Edge:** `chrome://extensions` → Режим разработчика →
     «Загрузить распакованное» → папка `chrome/` из релиза.
   - **Firefox:** `about:debugging` → «Загрузить временное дополнение» → `hydra-firefox.xpi`.
   - **Safari:** пока вручную через Xcode ([docs/SAFARI_SETUP.md](docs/SAFARI_SETUP.md)).
3. Готово — куки/сессия и многопоток работают сразу.

> Для распространения на чужие машины нужна нотаризация (Developer ID). Локально
> хватает ad-hoc подписи — `Hydra.app` запускается на своей машине без предупреждений.

## Сборка из исходников

Нужен Xcode/Swift toolchain и `rsvg` (для иконок: `brew install librsvg`).

```bash
./build-all.sh
```

Кладёт в `dist/`: `Hydra.app` (с вложенным хостом, само-регистрация), `chrome/`
(распакованное расширение), `hydra-chrome.zip`, `hydra-firefox.xpi`.

```bash
cd core && swift test          # тесты движка (байт-в-байт, резюм, санитайзинг имён)
.build/release/hydractl URL --out ~/Downloads --connections 8   # CLI
```

## Структура

```
core/Sources/
  DownloadCore/   движок: probe, блоки, резюм, сессия, лимит скорости, история
  HydraApp/       SwiftUI/AppKit: попап, окно, настройки, приёмник, уведомления
  hydractl/       CLI
  hydra-host/     native messaging host (делегирует в app, фолбэк — качает сам)
extension/        WebExtension (Chrome/Firefox), фиксированный key, попап-статус
app/build.sh      сборка Hydra.app (подпись, вложение host, иконка)
build-all.sh      ⭐ всё разом → dist/
docs/             ARCHITECTURE.md, SAFARI_SETUP.md, USAGE.md, screenshots/
```

## Расширение браузера

Тонкий слой: снимает сессию, перехватывает ссылки, показывает статус связи с
приложением и список текущих загрузок. Логику и UI несёт приложение. Настройки
перехвата (авто-перехват, мин. размер, типы файлов, потоки) — единый источник в
приложении, расширение читает их через хост.
