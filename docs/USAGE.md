# Hydra — Инструкция по использованию

## 1. UI приложения (Menu Bar)

После запуска `Hydra.app` в menu bar (справа вверху ↗️) появится иконка:
- **○ стрелка вниз** — нет активных загрузок
- **● стрелка вниз** (заполненная) — есть активные загрузки

### Как открыть окно управления:

**Кликни на иконку Hydra в menu bar** → откроется окно со списком загрузок.

В окне:
```
┌────────────────────────────────────┐
│ Hydra Downloads     2 active  Quit │  ← заголовок + кнопка выхода
├────────────────────────────────────┤
│ rfc2616.txt                        │
│ ████████████░░░ 80%  5.2 MB/s      │  ← прогресс + скорость
│                                    │
│ rfc9110.txt                        │
│ ████░░░░░░░░░░ 20%  3.8 MB/s       │
└────────────────────────────────────┘
```

Если загрузок нет:
```
┌────────────────────────────────────┐
│ Hydra Downloads     0 active  Quit │
├────────────────────────────────────┤
│          No downloads              │
│ Right-click a link in browser →    │
│      Download with Hydra           │
└────────────────────────────────────┘
```

## 2. Как скачать файл

### Через расширение браузера (рекомендуется):

1. **Установи расширение** (см. [README.md](../README.md))
2. **В браузере**: правый клик на ссылке → **"Download with Hydra"**
3. **Или**: перейди на прямую ссылку → браузер автоматически перехватит (если включено в настройках расширения)

### Через CLI:

```bash
/Users/pushi/Repos/Hydra/core/.build/release/hydractl \
  "https://example.com/file.zip" \
  --out ~/Downloads --connections 8
```

### Через IPC (для автоматизации):

```bash
echo '{"type":"download","url":"https://...","connections":8}' \
  | nc -U /tmp/hydra.sock
```

## 3. Настройки расширения

Кликни на иконку расширения → **Options** (или правый клик → "Параметры"):

- **Авто-перехват**: включить/выключить автоматический перехват всех загрузок
- **Минимальный размер**: не перехватывать файлы мельче N МБ
- **Типы файлов**: перехватывать только `.zip`, `.dmg`, `.iso`, и т.д.
- **Домены**: whitelist/blacklist конкретных доменов
- **Потоков на файл**: сколько параллельных соединений (по умолчанию 8)

## 4. Как это работает

```
Browser Extension
     ↓ (перехватывает download)
     ↓ (собирает Cookie, User-Agent, Referer)
     ↓
hydra-host (native messaging)
     ↓ (делегирует через Unix socket)
     ↓
Hydra.app (/tmp/hydra.sock)
     ↓ (качает в 8 потоков)
     ↓
~/Downloads/file.zip
```

## 5. Где файлы

По умолчанию: `~/Downloads/`

Если файл с таким именем уже существует → добавляется `(1)`, `(2)`, и т.д.

## 6. Выход из приложения

Два способа:
1. Кликни на иконку Hydra → кнопка **Quit** в правом верхнем углу
2. Cmd+Q (когда окно Hydra активно)

## 7. Проблемы

### Окно не открывается при клике на иконку
- Закрой и перезапусти `Hydra.app`
- Проверь: `ls -la /tmp/hydra.sock` (должен существовать)

### Расширение не перехватывает загрузки
- Проверь, что `hydra-host` зарегистрирован: `ls ~/Library/Application\ Support/Mozilla/NativeMessagingHosts/com.hydra.host.json`
- Перезапусти браузер
- Проверь консоль расширения (F12 → Console) на ошибки

### Загрузка идёт в один поток
- Сервер не поддерживает HTTP Range → нормально, Hydra автоматически падает в один поток
- Или токен в URL одноразовый → тоже один поток

### Авторизованные загрузки не работают
- Убедись, что расширение имеет доступ к кукам (permissions в манифесте)
- Cookie собираются для **финального URL после редиректов**
