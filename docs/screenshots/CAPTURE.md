# Как снять скриншоты для README

Приложение умеет демо-режим — наполняет себя выверенными моковыми загрузками
(качается с дефраг-сеткой, в очереди, пауза, ошибка + история). Реальные загрузки
при этом не трогаются.

## 1. Включить демо-режим и запустить

```bash
defaults write com.hydra.downloads demoMode -bool true
open dist/Hydra.app
```

## 2. Снять экраны (⌘⇧4, пробел — захват окна с тенью)

Сохраняйте в `docs/screenshots/` под этими именами — README уже на них ссылается:

| Файл | Что снять |
|---|---|
| `popover.png` | Поповер по клику на иконку в строке меню (список загрузок) |
| `window.png` | «Все загрузки» (иконка окна в шапке поповера) — список + история |
| `settings.png` | Настройки (шестерёнка) — вкладка «Загрузка» |
| `drop.png` | Плавающее окно-приёмник (включить в шапке поповера) |
| `menubar.png` | Кроп строки меню с иконкой-стрелкой и `%` (по желанию) |
| `extension.png` | Попап расширения в браузере (см. ниже) |
| `notification.png` | Баннер уведомления (скачайте реальный файл вне демо-режима) |

Попап расширения: перезагрузите расширение (`chrome://extensions` → ↻), откройте
попап — там список из приложения (демо-режим его тоже отдаёт через хост).

## 3. Выключить демо-режим

```bash
defaults delete com.hydra.downloads demoMode
```

Скриншоты ретины лучше уменьшить до ~2x ширины контента, чтобы не были гигантскими.
